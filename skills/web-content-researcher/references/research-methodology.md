# Research Methodology — Deep Reference

## Research Goal

Find content topics at the intersection of:
1. **Real search demand** — validated via web search results
2. **Product relevance** — connects to the user's ICP pain points, use cases, or competitive positioning

## Query Construction Guide

### General Rules
- Use natural language — search tools work best with how people actually search
- Include the current year for tools, trends, and comparisons
- Vary specificity — mix broad ("AI agent platforms for marketing") with narrow ("how to automate Smartlead campaigns with AI")
- Run 3–5 queries per category, 5 categories = 15–25 total queries per cycle

### Adapting Queries to Any Product

The SKILL.md provides example patterns. To adapt them:

1. **Read `.agents/product-marketing-context.md`** for the product's specific:
   - Pain points → Category 1 queries
   - Competitors → Category 2 queries
   - Product category/industry → Category 3 queries
   - Core workflows → Category 4 queries
   - Industry → Category 5 queries

2. **Replace placeholders** with actual product terms:
   - `[ICP pain point]` → the specific pain from the context doc
   - `[Competitor]` → actual competitor names
   - `[product category]` → the category from the context doc
   - `[industry]` → the industry vertical

### Example: SaaS Product for Marketing Agencies

If the product context says:
- ICP: Growth marketing agencies
- Pain: Manual client onboarding, thin margins
- Competitors: Clay, Instantly
- Category: AI agent platform

Then queries become:
- Cat 1: "how to scale marketing agency without hiring 2026"
- Cat 2: "Clay alternatives 2026", "Instantly vs Smartlead"
- Cat 3: "AI agents for marketing 2026"
- Cat 4: "how to automate lead enrichment"
- Cat 5: "AI agent platform news March 2026"

## Extraction Template

For each search result worth noting:

```
Query: [exact query]
Result Title: [title of top result]
URL: [source]
Content Type: [blog/video/social/docs/forum]
Key Themes: [2-3 themes]
Content Gap: [what's missing that the user's product could fill]
Search Intent: [informational/commercial/transactional]
Relevance: [High/Medium/Low]
Opportunity: [how to create better content]
```

## Filtering Criteria

### High Priority (Keep)
- Search demand evidence + direct product relevance
- Competitor comparison with genuine differentiator
- How-to with unique product angle
- Trending topic with authentic team expertise
- Content gap is real (not already well-covered)

### Low Priority (Deprioritize)
- Dominated by Wikipedia, major publications, or brand-owned SERPs
- No product or ICP connection
- Pure hype trending topics with nothing substantive to add
- Already thoroughly covered by the user's existing content
- Too niche (no evidence of search volume)

## Output Quality Checklist

Before handing off to the content-calendar skill:

- [ ] Every opportunity cites real search evidence
- [ ] Every opportunity connects to at least one ICP pain point or use case
- [ ] Mix of evergreen (blog/YouTube) and timely (LinkedIn) topics
- [ ] All 5 research categories represented
- [ ] No duplicate or overlapping topics
- [ ] Content gaps are genuine
- [ ] 15–20 opportunities compiled and ranked
